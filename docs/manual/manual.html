1. Introducción

Jaguata es una plataforma web que conecta Dueños con Paseadores verificados, permitiendo solicitar paseos, gestionar pagos y comprobantes, recibir notificaciones, administrar perfiles y utilizar un sistema de puntos y recompensas.

Objetivo

Facilitar la contratación de paseadores confiables, con control por roles, trazabilidad y reputación.

Seguridad

Verificación de paseadores mediante documentos, control administrativo, auditoría y estados de cuenta.

Recompensas

Se acumulan puntos por paseos y se canjean por beneficios (con reglas de uso).

2. Requisitos del sistema

Requisitos recomendados para que Jaguata funcione sin problemas.

Navegadores
  • Google Chrome (recomendado)
  • Microsoft Edge
  • Mozilla Firefox
Importante: habilitar JavaScript y permitir ventanas emergentes si el sistema descarga reportes.
Ubicación y mapas
  • Se recomienda permitir ubicación del navegador.
  • Para solicitar paseo, el dueño coloca un pin en el mapa (punto de retiro).
Tip: si no aparece el mapa, probá recargar la página y verificar conexión a internet.

Estas pantallas están disponibles sin iniciar sesión. Desde aquí el usuario puede conocer la plataforma, registrarse como Dueño o Paseador, aceptar bases y condiciones, iniciar sesión, recuperar contraseña y contactar soporte vía WhatsApp.

  • Botones de acceso rápido: Crear cuenta, Iniciar sesión y Contacto.
  • Información relevante para conocer Jaguata antes de registrarse.
¿Para qué sirve?
  • Explicar qué es Jaguata y cómo funciona.
  • Guiar al usuario al registro o inicio de sesión según necesidad.
  • Facilitar contacto rápido en caso de dudas.
¿Cómo se usa?
  1. Ingresar a la pantalla principal.
  2. Elegir una acción: Crear cuenta (registro) o Iniciar sesión (login).
  3. Si hay dudas o inconvenientes, presionar Contacto.
Crear cuenta → Registro Iniciar sesión → Login Contacto → WhatsApp soporte
Acceso 3.2 Crear cuenta (Registro)
¿Qué muestra?
  • Formulario de creación de cuenta.
  • Selección de rol: Dueño o Paseador.
  • Campos dinámicos según rol elegido.
Validación y seguridad
  • Si elige Paseador, se solicitan documentos para verificación.
  • Se requiere aceptar Bases y condiciones antes de completar el registro.
  • La cuenta queda en estado Pendiente hasta aprobación del Administrador.
Importante: el paseador debe cargar documentos (seguridad). El Administrador revisa y decide aprobar o rechazar la cuenta.

  5. Aceptar Bases y condiciones (pantalla correspondiente).
  6. Confirmar registro. El sistema deja la cuenta en pendiente para revisión.
Dueño: registro simple Paseador: documentos obligatorios Aceptar bases y condiciones

¿Qué muestra?
  • Términos y condiciones de uso de la plataforma.
  • Responsabilidades del usuario (Dueño/Paseador) y reglas del sistema.

  • Formalizar el uso correcto de la plataforma.
  • Establecer reglas de seguridad, pagos y comportamiento.

¿Cómo se usa?
  1. Leer el contenido.
  2. Marcar aceptación.
  3. Continuar el proceso de registro.
Acceso 3.4 Iniciar sesión (Login)
¿Qué muestra?
  • Formulario de acceso con usuario/correo y contraseña.
  • Opción de Recuperar contraseña.
  • Redirección automática al panel según rol (Dueño/Paseador/Admin).

  • Para ingresar como Dueño o Paseador, la cuenta debe estar Aprobada y Activa.
  • El Administrador revisa el registro y habilita el acceso.

¿Cómo se usa? (pasos)
  1. Ingresar a Login.
  2. Completar credenciales.
  3. Presionar Ingresar.
  4. Si la cuenta está activa, el sistema abre el panel del rol correspondiente.
  5. Si la cuenta está pendiente/rechazada/suspendida, el sistema bloquea el acceso.
Seguridad 3.5 Recuperación de contraseña
¿Qué permite?
  • Recuperar acceso si el usuario olvidó su contraseña.
  • Seguir pasos guiados (según el flujo implementado en el sistema).
Recomendación
  • Usar contraseñas seguras (mínimo 8 caracteres, letras y números).
  • No compartir credenciales.
Si en tu instalación local no existe la tabla/flujo (por ejemplo password_resets), la recuperación puede estar deshabilitada o en desarrollo.
Soporte 3.6 Contacto (Soporte por WhatsApp)
¿Qué muestra?
  • Listado de consultas frecuentes o categorías de problemas.
  • Al seleccionar una consulta, se abre WhatsApp con un mensaje prearmado.
¿Para qué sirve?
  • Facilitar soporte rápido al usuario.
  • El Administrador revisa el caso y responde indicando el conflicto y la solución.
  • Se usa especialmente cuando hay: rechazo de suscripción, problemas de acceso o dudas.
¿Cómo se usa? (pasos)
  1. Ingresar a Contacto desde la pantalla pública principal.
  2. Elegir la consulta que más se ajuste al problema.
  3. El sistema abre WhatsApp y envía el mensaje al soporte.
  4. El Admin verifica el incidente y responde al usuario.
Cuándo usar Contacto:
  • Cuenta pendiente / no aprobada.
  • Suscripción rechazada o pago no validado.
  • Problemas de acceso o funcionamiento.
Resumen de flujo público:
  1. Sobre nosotros → usuario entiende el sistema.
  2. Registro → elige rol (Dueño/Paseador) + documentos si es paseador.
  3. Bases y condiciones → aceptación obligatoria.
  4. Login → acceso solo si Admin aprobó/activó cuenta.
  5. Contacto → soporte por WhatsApp para resolver incidentes.

4. Navegación general

Una vez que el usuario inicia sesión, Jaguata muestra un panel privado según su rol: Administrador, Dueño o Paseador. Todos los paneles comparten la misma lógica: un menú lateral (sidebar), contenido central y notificaciones.

Sidebar Menú lateral (Desktop)
  • Muestra los módulos disponibles para el rol.
  • Resalta la pantalla actual (link active).
  • Permite acceder a reportes, acciones y configuraciones.
Mobile Topbar + menú desplegable
  • En pantallas pequeñas aparece la barra superior (“Jaguata” + botón ☰).
  • Al tocar el botón, el sidebar se abre con overlay.
  • Se cierra al tocar fuera, al elegir una opción o al presionar ESC.
Tip de uso: si estás en celular y el menú queda abierto, tocá fuera (overlay) para cerrarlo. El sistema también lo cierra automáticamente al entrar a una pantalla.

4.1 Sidebars por rol

Admin Sidebar Administrador

Incluye accesos a: Inicio, Usuarios, Mascotas, Paseos, Pagos, Suscripciones, Notificaciones, Reporte de Ganancias, Auditoría y Configuración. También incorpora cierre de sesión.

Dueño Sidebar Dueño

Incluye submenús colapsables para: Reservas (Solicitar Paseo, Mis Paseos, Buscar Paseadores), Pagos (Gastos Totales, Pagos), Mascotas (Mis Mascotas, Agregar), Perfil (Mi Perfil, Editar), Puntos & Recompensas (Mis Puntos) y Configuración.

Paseador Sidebar Paseador

Incluye accesos a: Inicio, Solicitudes, Mis Paseos, Remuneraciones, Notificaciones, Suscripción PRO (con badge de estado), Perfil (Mi Perfil, Editar) y Configuración.

4.2 Elementos comunes en todas las pantallas

Filtros y búsqueda
  • Listas con buscador para encontrar registros rápidamente.
  • Filtros por estado (pendiente, aprobado, etc.) y fechas (según módulo).
  • Exportación a Excel cuando el módulo lo permite.
Notificaciones
  • Avisos del sistema (acciones, estados, promociones, recordatorios).
  • En Dueño/Paseador: muestra comunicados del Administrador.
  • En Admin: permite crear y enviar notificaciones masivas.

5. Rol Administrador

El Administrador controla la operación completa de Jaguata: aprobación de cuentas, control de seguridad, supervisión de paseos y pagos, gestión de suscripciones, envío de notificaciones y auditoría.

Objetivo del rol: garantizar la seguridad y el correcto funcionamiento del sistema, protegiendo a dueños, paseadores y mascotas.
Admin 5.1 Dashboard del Administrador
¿Qué muestra?
  • Estadísticas generales del sistema (usuarios, paseos, pagos, etc.).
  • Tarjetas (cards) que resumen el movimiento de Dueños y Paseadores.
  • Indicadores de calificaciones/reputación y actividad reciente.
¿Para qué sirve?
  • Dar al Admin una vista rápida del estado del sistema.
  • Detectar situaciones anormales: muchas cancelaciones, reportes, etc.
  • Acceso rápido a módulos de control (usuarios, pagos, auditoría).
¿Cómo se usa?
  1. Ingresar al Dashboard luego del login del Administrador.
  2. Revisar tarjetas de resumen para identificar alertas.
  3. Entrar al módulo correspondiente desde el menú lateral.
Admin 5.2 Gestión de Usuarios
¿Qué muestra?
  • Listado de usuarios registrados (Dueños y Paseadores).
  • Estado de cuenta: pendiente, aprobado, rechazado, suspendido, activo/inactivo.
  • Información de suscripción (cuando aplica para paseadores).
  • Acciones: aprobar, rechazar, suspender o activar.
¿Para qué sirve?
  • Validar identidad del usuario (especialmente paseadores).
  • Aplicar control por reportes: suspender si infringe normas.
  • Generar reporte Excel completo con reputación y comentarios.
Seguridad: en editar_usuario.php el Admin puede ver documentos del paseador (cédula, antecedente, selfie con cédula) para aprobar con mayor confianza.
¿Cómo se usa?
  1. Entrar a Usuarios.
  2. Buscar el usuario por filtros/buscador.
  3. Revisar su estado y, si corresponde, aprobar o rechazar.
  4. En el botón de lápiz, entrar al perfil para ver documentos y más datos.
  5. Exportar Excel si necesita un reporte general.

5.3 Gestión de Mascotas

Este módulo permite al Administrador supervisar las mascotas registradas por los dueños, para asegurar que los datos sean correctos y mantener control ante reportes o irregularidades.

Admin Listado de Mascotas
¿Qué contiene?
  • Listado de mascotas registradas por los dueños.
  • Datos principales: nombre, especie/raza (si aplica), edad, dueño asociado, estado.
  • Acciones administrativas: desactivar mascotas reportadas o incorrectas.
  • Botón para exportar Excel (reporte total de mascotas).
¿Para qué sirve?
  • Control y trazabilidad del contenido del sistema.
  • Medida de seguridad ante reportes de dueños/paseadores.
  • Generación de reportes para anexos o auditoría interna.
¿Cómo se usa?
  1. Entrar a Mascotas desde el menú del Administrador.
  2. Buscar por nombre, dueño o filtros de estado.
  3. Si existe un reporte o inconsistencia, seleccionar Desactivar.
  4. Exportar Excel cuando se requiera un reporte global.
Tip: desactivar una mascota es útil si el dueño la registró con datos erróneos o si hay un conflicto con paseos asociados.

5.4 Gestión de Paseos

El Administrador puede monitorear todos los paseos del sistema. Esto permite verificar estados, controlar incidentes y actuar ante reportes (cancelar/finalizar si corresponde).

Admin Listado de Paseos
¿Qué contiene?
  • Listado visual de paseos con su estado (pendiente/aceptado/en curso/completo/cancelado).
  • Filtros y búsqueda por dueño, paseador, fecha o estado (según tu implementación).
  • Acción Ver para ingresar al detalle del paseo.
  • Exportación a Excel (reporte de paseos del sistema).
¿Qué se ve en “VerPaseo”?
  • Mapa con ubicación (pin) y detalles del punto de retiro.
  • Datos del paseo: dueño, paseador, mascota(s), fecha, duración, total.
  • Acciones administrativas: finalizar o cancelar (cuando corresponde).
Importante: cancelar/finalizar desde Admin se usa solo cuando existe un incidente, reporte o verificación necesaria. La auditoría registra estas acciones.
¿Cómo se usa?
  1. Entrar a Paseos.
  2. Filtrar por estado (por ejemplo: pendientes o en curso).
  3. Presionar Ver para analizar el detalle (mapa + información completa).
  4. Si corresponde por política del sistema, Finalizar o Cancelar.
  5. Exportar Excel cuando se requiera un reporte.

5.5 Gestión de Pagos

El Administrador supervisa los pagos registrados en el sistema, verificando su estado y su relación con un paseo. Esto garantiza transparencia y control financiero.

Admin Listado de Pagos
¿Qué contiene?
  • Listado de pagos con estados (pendiente/confirmado/rechazado según tu lógica).
  • Datos clave: dueño, monto, método (transferencia/efectivo), fecha.
  • Acciones: ver detalle, descargar reporte Excel.
¿Qué se ve en “DetallePago”?
  • Información completa del pago.
  • Comprobante adjunto (si fue transferencia).
  • Botón para abrir el paseo asociado (VerPaseo).
Transparencia: el detalle vinculado al paseo permite validar que el pago corresponde al servicio realizado (mascotas, duración, total).
¿Cómo se usa?
  1. Entrar a Pagos.
  2. Filtrar por estado si se necesita revisar pendientes.
  3. Presionar Ver para entrar a DetallePago.
  4. Revisar comprobante y, si hace falta, abrir el paseo asociado en VerPaseo.
  5. Exportar Excel para auditoría o anexos.

5.6 Gestión de Suscripciones

Este módulo permite al Administrador gestionar las suscripciones de los paseadores, validando los pagos realizados para habilitar o rechazar el acceso profesional al sistema.

Admin Suscripciones Pendientes
¿Qué muestra esta pantalla?
  • Listado de paseadores que solicitaron suscripción.
  • Estado de la suscripción: pendiente, aprobada o rechazada.
  • Visualización del comprobante de pago adjunto.
  • Acciones para Aprobar o Rechazar la suscripción.
¿Para qué sirve?
  • Garantizar que solo paseadores verificados accedan al sistema.
  • Controlar el ingreso económico por suscripciones.
  • Evitar accesos sin pago confirmado.
Importante: si la suscripción es rechazada, el paseador deberá contactar al soporte desde la pantalla pública de contacto para aclarar el inconveniente.

5.7 Centro de Notificaciones

Desde este módulo el Administrador puede enviar notificaciones informativas, promocionales o técnicas a los usuarios del sistema.

Admin Envío de Notificaciones

Uso recomendado: notificar mantenimientos, cambios de tarifas, campañas promocionales o avisos de seguridad.

Admin Registro de Actividades

  • Eventos críticos del sistema.

Admin Parámetros Generales

6. Rol Dueño

El Dueño registra sus mascotas, busca paseadores, solicita paseos, realiza pagos, revisa notificaciones y utiliza puntos y recompensas.

Módulos del Dueño: Dashboard, Solicitar Paseo, Mis Paseos, Buscar Paseadores, Gastos Totales, Pagos/Comprobantes, Mascotas, Notificaciones, Perfil, Puntos y Configuración.

6.1 Dashboard (Inicio del Dueño)

Es la primera pantalla que ve el dueño al ingresar. Aquí se resume todo lo importante: paseos recientes, gastos, mascotas registradas, notificaciones y accesos rápidos.

Dueño Panel principal
¿Qué contiene?
  • Mensaje de bienvenida al usuario logueado.
  • Tarjetas (cards) con resúmenes: paseos, gastos, mascotas y notificaciones.
  • Sección de paseos recientes para ver el movimiento actual.
  • Accesos directos a módulos principales (Solicitar Paseo, Mis Mascotas, Pagos).
¿Para qué sirve?
  • Permite al dueño controlar rápidamente su estado en el sistema.
  • Facilita el acceso a acciones frecuentes sin recorrer todo el menú.
  • Sirve como “resumen del día” y punto de partida para solicitar paseos.
Tip: si ves una notificación importante (promos, avisos o cambios), ingresá al módulo Notificaciones para leer el detalle.

6.2 Solicitar Paseo

Pantalla principal para crear una solicitud. Funciona como un “asistente por pasos” para elegir mascota(s), ubicación, fecha/hora, paseador disponible y aplicar recompensas.

Dueño Asistente de reserva
¿Cómo funciona?
  1. Seleccionás Mascota 1 (obligatoria).
  2. Opcional: seleccionás Mascota 2 (máximo 2 y no repetidas).
  3. Marcás tu ubicación en el mapa (pin) o permitís ubicación del navegador.
  4. Elegís fecha, hora y datos del paseo (duración/otros campos si aplica).
  5. El sistema propone el paseador más cercano y disponible.
  6. Confirmás la solicitud y se envía al paseador.
Recompensas y puntos
  • Al final, el sistema permite usar recompensa (si el dueño canjeó puntos).
  • La recompensa se valida desde Mis Puntos.
  • Las recompensas aplican al total y cambian según la promoción canjeada.
Regla clave del sistema: si seleccionás 2 mascotas, se aplica automáticamente un 30% de descuento.
Requisito: para solicitar un paseo debés tener al menos 1 mascota registrada. Si no aparecen mascotas, primero ingresá a Mascotas → Agregar Mascota.

6.3 Mis Paseos

Permite al dueño ver el historial completo de solicitudes y paseos. Incluye cuadros de resumen, filtros y acciones para ver detalle, cancelar o calificar según el estado.

Dueño Historial y seguimiento
¿Qué muestra?
  • Cuadros de resumen: cantidad de paseos, pendientes, completados, cancelados (según tu UI).
  • Lista/tabla con paseos del dueño y su estado.
  • Filtros para buscar por fecha/estado (según implementación).
  • Acción Ver para abrir el detalle del paseo.
Acciones según estado
  • Pendiente/Asignado: se puede cancelar (si tu regla lo permite).
  • En curso: seguimiento del estado (según tu flujo).
  • Completado: permite calificar al paseador.
Detalle del paseo: desde el botón “Ver” se ingresa a una pantalla con mapa, información completa y acciones disponibles.
Dueño Ver Paseo (Detalle)
¿Qué contiene?
  • Mapa con la ubicación del paseo.
  • Datos del dueño, mascota(s) y paseador asignado.
  • Estado del paseo (pendiente, en curso, completado, cancelado).
¿Qué se puede hacer?
  • Cancelar paseo (si aún no fue completado y cumple reglas).
  • Calificar paseo cuando finaliza.
  • Visualizar un resumen claro para respaldo.

6.4 Buscar Paseadores

En esta pantalla el dueño puede ver paseadores disponibles por zona y revisar su perfil antes de solicitar un paseo. Es ideal para comparar reputación, comentarios y experiencia.

Dueño Paseadores disponibles por zona
¿Qué contiene?
  • Listado/tabla o tarjetas con paseadores cercanos (según tu diseño).
  • Datos resumidos: nombre, zona, reputación promedio, disponibilidad.
  • Botón Ver perfil para ver información completa.
¿Para qué sirve?
  • Permite elegir un paseador con mejor reputación o más cercano.
  • Ayuda a verificar opiniones antes de reservar.
  • Reduce riesgos al contratar, porque se muestra historial y comentarios.
Tip: si no aparecen paseadores, probá seleccionar una ubicación válida o mover el pin del mapa al solicitar paseo (esto ayuda a detectar la zona correctamente).
Dueño Ver Paseador (Perfil)
Información relevante
  • Reputación (promedio de estrellas).
  • Opiniones y comentarios de otros dueños.
  • Zona de trabajo y disponibilidad (si tu vista lo incluye).
¿Qué se puede hacer?
  • Evaluar al paseador antes de reservar.
  • Volver a la solicitud y confirmar el paseo con más confianza.

6.5 Gastos Totales

Muestra el acumulado de pagos realizados por el dueño logueado. Sirve para control financiero, seguimiento de consumo y respaldo para reportes.

Dueño Resumen financiero del dueño
¿Qué contiene?
  • Resumen del total gastado (por periodo o total general según tu UI).
  • Listado de pagos asociados a paseos.
  • Opción de descargar/exportar si está habilitada.
¿Para qué sirve?
  • Controlar cuánto se invirtió en paseos.
  • Verificar pagos y evitar confusiones en transferencias.
  • Generar reportes para uso personal o respaldo de tesis.

6.6 Pagos

Lista todos los pagos realizados por el dueño, incluyendo comprobantes cuando se paga por transferencia. Desde aquí se puede acceder al comprobante detallado.

Dueño Historial de pagos y comprobantes
¿Qué contiene?
  • Tabla/listado de pagos con datos principales (monto, fecha, estado, método).
  • Indicador de comprobante adjunto cuando es transferencia.
  • Acción para abrir detalle del comprobante.
¿Para qué sirve?
  • Verificar que el pago fue registrado correctamente.
  • Guardar evidencia y evitar rechazos por comprobantes incompletos.
  • Respaldar cada pago con su archivo adjunto.
Importante: si el comprobante está borroso o incompleto, el pago puede quedar en revisión. Siempre subí una imagen clara o PDF legible.
Dueño Comprobante de pago (Detalle)
¿Qué se ve?
  • Datos del pago: monto, fecha, método y estado.
  • Archivo adjunto (comprobante) para abrir o descargar.
  • Relación con el paseo/pedido asociado (si tu vista lo muestra).
Acciones
  • Abrir el comprobante en una nueva pestaña.

8. Preguntas frecuentes

Respuestas rápidas a dudas comunes de Dueños, Paseadores y Administrador.

Registro y acceso
  • ¿Por qué no puedo iniciar sesión? La cuenta puede estar pendiente, rechazada o suspendida.
  • ¿Cuándo se habilita mi cuenta? Cuando el Administrador revisa y aprueba la solicitud.
  • ¿Qué pasa si soy paseador? Debés cargar documentos y esperar validación. Además, si tu sistema lo exige, necesitás Suscripción PRO.
Paseos y estados
  • ¿Qué estados puede tener un paseo? Pendiente / Aceptado / En curso / Completado / Cancelado (según tu flujo).
  • ¿Por qué no aparece “Iniciar paseo”? Puede depender del estado, horario permitido o confirmación del pedido.
  • ¿Puedo cancelar un paseo? Sí, si tu regla lo permite y el paseo no fue completado (depende del estado).
Puntos y recompensas
  • ¿Cómo gano puntos? Al completar paseos (según la regla del sistema).
  • ¿Cuántas veces puedo canjear? En tu manual: 1 vez por día.
  • ¿Dónde aplico la recompensa? En Solicitar Paseo, al final del proceso.
Regla clave: si seleccionás 2 mascotas se aplica automáticamente 30% de descuento.
Pagos y comprobantes
  • ¿Qué pasa si el comprobante es ilegible? Puede quedar en revisión o rechazo.
  • ¿Dónde veo mi comprobante? Dueño → Pagos → Comprobante.
  • ¿Cómo evita errores? Subí imágenes claras o PDF legible, con monto y fecha visibles.

El correcto envío de comprobantes garantiza que los pagos sean validados rápidamente y evita demoras en la confirmación de los paseos.
